## Overview

JAL (Java Assembly Language) is a domain-specific language that enables developers to write Java Virtual Machine (JVM)
bytecode in a human-readable, assembly-like syntax. It provides a direct mapping to Java class files while maintaining
readability through high-level constructs like labels, field references, and method calls.

Unlike raw bytecode, JAL abstracts away the complexity of constant pools and bytecode offsets, allowing programmers to
focus on algorithm logic at the bytecode level.

---

## Basic Structure

A JAL source file contains a single class or interface definition:

```
JAL File
  └── Class/Interface Definition
        ├── Class Metadata (version, superclass, interfaces)
        ├── Fields
        └── Methods
```

### Minimal Example

```jal
public class HelloWorld (
  major_version=55,
  minor_version=0) {
  
  public static main([Ljava/lang/String;)V {
    return
  }
}
```

## Lexical Elements

### Keywords

#### Class Declaration

- `class` - Declares a class
- `interface` - Declares an interface

#### Access Modifiers

- `public` - Accessible from any location
- `private` - Accessible only within the class
- `protected` - Accessible within the package and subclasses

#### Class-level Attributes

- `static` - Belongs to the class, not instances
- `final` - Cannot be extended or overridden
- `abstract` - Must be implemented by subclasses
- `super` - Marks the superclass
- `synthetic` - Generated by the compiler
- `annotation` - Indicates an annotation type
- `enum` - Indicates an enumeration
- `strictfp` - Floating-point arithmetic follows IEEE 754 exactly

#### Method Attributes

- `synchronized` - Thread-safe execution
- `bridge` - Generated bridge method
- `varargs` - Accepts variable number of arguments
- `native` - Implemented in native code

#### Field Attributes

- `volatile` - Value may change unexpectedly
- `transient` - Not included in serialization

#### Class Properties (in metadata)

- `major_version` - Java version (e.g., 55 = Java 11, 52 = Java 8)
- `minor_version` - Class file version minor number
- `super_class` - The parent class
- `interfaces` - Implemented interfaces

#### Special Method Names

- `<init>` - Constructor
- `<clinit>` - Static initializer

### Literals

#### Numbers

```jal
42              // Decimal integer
-123            // Negative integer
0xFF            // Hexadecimal
3.14            // Floating-point
100L            // Long integer
3.14f           // Float
```

#### Booleans

```jal
true
false
```

#### Strings

```jal
"Hello, World!"
'Single quotes work too'
"Escape sequences: \n \t \\ \" \'"
```

#### Identifiers

Identifiers must start with a letter, dollar sign (`$`), or underscore (`_`), followed by zero or more letters, digits,
dollar signs, or underscores:

```jal
myVariable
_privateField
$dollarField
ClassName
method1Name
```

### Comments

```jal
// Single-line comment

/* Multi-line
   comment
   spanning several lines */
```

### Operators and Symbols

| Symbol  | Name               | Usage                                        |
|---------|--------------------|----------------------------------------------|
| `->`    | Reference operator | `class -> field:Type`, `method -> name(...)` |
| `~`     | Label marker       | Used in exception directives `[~ label]`     |
| `:`     | Colon              | Label definition, type separator             |
| `=`     | Assignment         | Field initialization, metadata assignment    |
| `;`     | Semicolon          | Optional instruction terminator              |
| `(` `)` | Parentheses        | Method descriptors, metadata grouping        |
| `{` `}` | Braces             | Class/method body, switch cases              |
| `[` `]` | Brackets           | Array types, local variable info             |
| `,`     | Comma              | Separator for lists                          |
| `/`     | Slash              | Package/class separator in names             |

## Class Definition

### Syntax

```antlr
classDefinition : accModClass (KWD_CLASS | KWD_INTERFACE) className 
                  (LP classMeta? RP)? LBR classBody RBR;
```

### Structure

```jal
[AccessModifier] [ClassAttributes] class|interface ClassName 
  (ClassMetadata)? {
    FieldDefinitions
    MethodDefinitions
}
```

### Example

```jal
public class Student (
  major_version=55,
  minor_version=0,
  super_class=java/lang/Object,
  interfaces=java/lang/Cloneable) {
  
  // Fields go here
  // Methods go here
}
```

### Class Metadata

Class metadata is optional and enclosed in parentheses:

```jal
(major_version=55, minor_version=0, super_class=BaseClass, 
 interfaces=Serializable, Cloneable)
```

#### major_version

Specifies the JVM class file version:

| Version | Java Release |
|---------|--------------|
| 52      | Java 8       |
| 53      | Java 9       |
| 54      | Java 10      |
| 55      | Java 11      |
| 61      | Java 17      |

```jal
public class MyClass (major_version=55) {
  // For Java 11
}
```

#### super_class

Defines the parent class. Default is `java/lang/Object`:

```jal
public class Child (super_class=Parent) {
  // Extends Parent
}
```

#### interfaces

Lists all implemented interfaces:

```jal
public class Worker (
  interfaces=Runnable,Serializable) {
  // Implements multiple interfaces
}
```

## Fields

### Syntax

```antlr
fieldDefinition : accModField fieldName COLON typeDescriptor 
                  (EQ jvmInsArgScalarType)?;
```

### Structure

```jal
[AccessModifier] [FieldAttributes] fieldName : TypeDescriptor [= InitialValue];
```

### Examples

```jal
public count : I;

private phoneNumber : Ljava/lang/String;;

static final PI : D = 3.14;

volatile current : I = 0;
```

### Field Attributes

```jal
static      // Class variable (shared among all instances)
final       // Constant, cannot be modified after initialization
volatile    // Indicates value may change unexpectedly
transient   // Excluded from serialization
enum        // Enumeration constant
synthetic   // Generated by compiler
```

### Initialization

Fields can be initialized with scalar values:

```jal
private age : I = 25;
public name : Ljava/lang/String; = "John";
static count : J = 100;
final PI : D = 3.14159;
```

## Methods

### Syntax

```antlr
methodDefinition : accModMethod methodName methodDescriptor methodBody;
methodName : ID | KWD_MNAME_INIT | KWD_MNAME_CLINIT;
methodDescriptor : METHOD_DESCRIPTOR_ARG typeDescriptor;
methodBody : LBR instructionSet* RBR;
```

### Structure

```jal
[AccessModifier] [MethodAttributes] methodName (ParameterTypes)ReturnType {
  Instructions
}
```

### Examples

#### Regular Method

```jal
public add (II)I {
  iload_0
  iload_1
  iadd
  ireturn
}
```

This method:

- Takes two integers as parameters
- Returns an integer
- Loads first param, loads second param, adds them, returns result

#### Constructor

```jal
public <init> ()V {
  aload_0
  invokespecial java/lang/Object -> <init> ()V
  return
}
```

#### Static Initializer

```jal
static <clinit> ()V {
  iconst_0
  putstatic MyClass -> counter : I
  return
}
```

### Method Descriptors

Method descriptors define parameter types and return type in JVM format:

```
(ParameterTypes)ReturnType
```

#### Common Descriptors

```jal
()V              // No parameters, void return
(I)I             // One int parameter, int return
(II)I            // Two int parameters, int return
(Ljava/lang/String;)I  // String parameter, int return
([I)I            // Int array parameter, int return
([Ljava/lang/String;)V // String array parameter, void return
```

### Method Attributes

```jal
public        // Accessible from anywhere
private       // Accessible only within class
protected     // Accessible within package and subclasses
static        // Class method, no instance reference
final         // Cannot be overridden
abstract      // Must be implemented by subclasses
synchronized  // Thread-safe
native        // Implemented in native code
strict        // Strict floating-point behavior
```

## Type Descriptors

Type descriptors represent Java types in JVM format:

### Primitive Types

| Descriptor | Type    | Range/Size                |
|------------|---------|---------------------------|
| `V`        | void    | N/A                       |
| `Z`        | boolean | true/false                |
| `B`        | byte    | -128 to 127               |
| `C`        | char    | 0 to 65535                |
| `S`        | short   | -32768 to 32767           |
| `I`        | int     | -2³¹ to 2³¹-1             |
| `J`        | long    | -2⁶³ to 2⁶³-1             |
| `F`        | float   | IEEE 754 single precision |
| `D`        | double  | IEEE 754 double precision |

### Object Types

Object types are written as `L` followed by the fully qualified class name, ending with `;`:

```jal
Ljava/lang/Object;
Ljava/lang/String;
Ljava/util/ArrayList;
Ljava/io/PrintStream;
```

Your custom classes:

```jal
Lcom/example/MyClass;
Lcom/example/package/SomeClass;
```

### Array Types

Arrays are represented with `[` prefix:

```jal
[I              // int[]
[Z              // boolean[]
[[I             // int[][]
[Ljava/lang/String;   // String[]
[[[D            // double[][][]
```

### Examples in Context

```jal
// Field declarations with various types
public numbers : [I;
private name : Ljava/lang/String;;
static PI : D = 3.14;
volatile flag : Z;
public entities : [Ljava/lang/Object;;

// Method descriptors
main([Ljava/lang/String;)V  // main(String[] args)
sort([I)V                   // sort(int[] arr)
equals(Ljava/lang/Object;)Z // equals(Object obj)
```

## Instructions

JAL supports over 200 JVM bytecode instructions organized by category.

### Stack Operations

#### Push Constants

```jal
aconst_null                  // Push null
iconst_0 ... iconst_5       // Push integer 0 to 5
iconst_m1                   // Push -1
lconst_0, lconst_1         // Push long 0 or 1
fconst_0, fconst_1, fconst_2  // Push float 0, 1, or 2
dconst_0, dconst_1         // Push double 0 or 1
bipush 100                  // Push byte (-128 to 127)
sipush 10000                // Push short (-32768 to 32767)
ldc "Hello"                 // Push string/constant
ldc_w 65535                 // Push wide constant
```

**Example:**

```jal
public getZero ()I {
  iconst_0
  ireturn
}

public getMessage ()Ljava/lang/String; {
  ldc "Hello, World!"
  areturn
}
```

#### Load Local Variables

```jal
iload_0 ... iload_3        // Load int from local 0-3
iload 5                    // Load int from local 5
lload_0 ... lload_3        // Load long from local 0-3
fload_0 ... fload_3        // Load float from local 0-3
dload_0 ... dload_3        // Load double from local 0-3
aload_0 ... aload_3        // Load object from local 0-3
aload 10                   // Load object from local 10
```

**Example:**

```jal
public addTwo (II)I {
  iload_0          // Load first int parameter
  iload_1          // Load second int parameter
  iadd             // Add them
  ireturn          // Return result
}
```

#### Store Local Variables

```jal
istore_0 ... istore_3       // Store int to local 0-3
istore 5                    // Store int to local 5
lstore_0 ... lstore_3       // Store long to local 0-3
fstore_0 ... fstore_3       // Store float to local 0-3
dstore_0 ... dstore_3       // Store double to local 0-3
astore_0 ... astore_3       // Store object to local 0-3
astore 10                   // Store object to local 10
```

**Example with Local Variable Info:**

```jal
public setCounter (I)V {
  iload_1 [->counter]     // Load parameter with variable name hint
  istore_2 [I]            // Store with type hint
  return
}
```

#### Stack Manipulation

```jal
dup              // Duplicate top stack item
dup_x1           // Duplicate top and insert below second
dup_x2           // Duplicate top and insert below third
dup2             // Duplicate two top stack items
dup2_x1          // Duplicate two and insert below third
dup2_x2          // Duplicate two and insert below fourth
swap             // Swap top two stack items
pop              // Remove top stack item
pop2             // Remove two top stack items
```

**Example:**

```jal
public printTwice ()V {
  ldc "Message"
  dup              // Duplicate the string
  astore_1         // Store one copy
  aload_1          // Load it back
  // Now we have two copies on stack
  return
}
```

### Arithmetic Operations

```jal
// Integer arithmetic
iadd, isub, imul, idiv, irem       // +, -, *, /, %
ineg                               // Negate
ishl, ishr, iushr                  // <<, >>, >>>

// Long arithmetic
ladd, lsub, lmul, ldiv, lrem, lneg

// Float arithmetic
fadd, fsub, fmul, fdiv, frem, fneg

// Double arithmetic
dadd, dsub, dmul, ddiv, drem, dneg

// Bitwise operations
iand, ior, ixor                    // &, |, ^
land, lor, lxor                    // (for long)
```

**Example:**

```jal
public factorial (I)I {
  iconst_1        // result = 1
  istore_2
  iload_0         // Load parameter
  istore_1        // i = parameter
  
  loop:
  iload_1         // Load i
  ifle loopEnd     // if i <= 0, exit
  
  iload_2         // Load result
  iload_1         // Load i
  imul            // Multiply
  istore_2        // Store back to result
  
  iinc 1 -1       // i--
  goto loop
  
  loopEnd:
  iload_2         // Return result
  ireturn
}
```

### Type Conversions

```jal
i2l, i2f, i2d              // int to long, float, double
l2i, l2f, l2d              // long to int, float, double
f2i, f2l, f2d              // float to int, long, double
d2i, d2l, d2f              // double to int, long, float
i2b, i2c, i2s              // int to byte, char, short
```

**Example:**

```jal
public doubleToInt (D)I {
  dload_0        // Load double parameter
  d2i            // Convert to int
  ireturn
}
```

### Array Operations

#### Array Creation

```jal
newarray I              // Create int array
newarray Z              // Create boolean array
anewarray Ljava/lang/String;  // Create object array
multianewarray [I 2     // Create 2D array
```

#### Array Access

```jal
iaload, iastore        // int array load/store
laload, lastore        // long array load/store
faload, fastore        // float array load/store
daload, dastore        // double array load/store
aaload, aastore        // object array load/store
baload, bastore        // byte array load/store
caload, castore        // char array load/store
saload, sastore        // short array load/store
arraylength            // Get array length
```

**Example: BubbleSort (from examples)**

```jal
public static main([Ljava/lang/String;)V {
  // int[] arr = {5, 3, 8, 4, 2};
  iconst_5
  newarray I           // Create array of 5 ints
  dup
  iconst_0
  iconst_5
  iastore              // arr[0] = 5
  dup
  iconst_1
  iconst_3
  iastore              // arr[1] = 3
  // ... more array initialization
  astore_1             // Store array reference
  
  // Later: access array
  aload_1              // Load array
  iload_2              // Load index
  iaload               // Load element from array
  return
}
```

### Field Access

```jal
getfield ClassName -> fieldName : FieldType
putfield ClassName -> fieldName : FieldType
getstatic ClassName -> fieldName : FieldType
putstatic ClassName -> fieldName : FieldType
```

**Examples:**

```jal
// Print System.out
getstatic java/lang/System -> out : Ljava/io/PrintStream;

// Store to instance field
putfield MyClass -> count : I

// Access static field
getstatic MyClass -> totalCount : I
```

### Method Invocation

```jal
invokevirtual ClassName -> methodName MethodDescriptor
invokespecial ClassName -> methodName MethodDescriptor
invokestatic ClassName -> methodName MethodDescriptor
invokeinterface ClassName -> methodName MethodDescriptor
invokedynamic methodName MethodDescriptor ...
```

**Examples from HelloWorld.jal:**

```jal
// System.out.println("Hello, World!")
getstatic java/lang/System -> out : Ljava/io/PrintStream;
ldc "Hello, World!"
invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
```

```jal
// ArrayList.add(element)
invokevirtual java/util/ArrayList -> add (Ljava/lang/Object;)Z

// Math.sqrt(double) - static method
invokestatic java/lang/Math -> sqrt (D)D

// Interface method
invokeinterface java/util/List -> add (Ljava/lang/Object;)Z
```

### Return Instructions

```jal
ireturn             // Return int
lreturn             // Return long
freturn             // Return float
dreturn             // Return double
areturn             // Return object reference
return              // Return void
```

**Examples:**

```jal
public getAge ()I {
  bipush 25
  ireturn
}

public getName ()Ljava/lang/String; {
  ldc "Alice"
  areturn
}

public printMessage ()V {
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "Done"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  return
}
```

## Control Flow

### Branching

#### Conditional Branches (Single Value)

```jal
ifeq label          // if equal to 0
ifne label          // if not equal to 0
iflt label          // if less than 0
ifle label          // if less than or equal to 0
ifgt label          // if greater than 0
ifge label          // if greater than or equal to 0
ifnull label        // if null (for objects)
ifnonnull label     // if not null (for objects)
```

#### Comparison Branches (Two Values)

```jal
if_icmpeq label     // if int1 == int2
if_icmpne label     // if int1 != int2
if_icmplt label     // if int1 < int2
if_icmple label     // if int1 <= int2
if_icmpgt label     // if int1 > int2
if_icmpge label     // if int1 >= int2
if_acmpeq label     // if object1 == object2
if_acmpne label     // if object1 != object2
```

#### Unconditional Jump

```jal
goto label          // Jump to label
goto_w label        // Jump far
```

**Example: FizzBuzz (from examples)**

```jal
public static main([Ljava/lang/String;)V {
  iconst_1
  istore_1           // i = 1
  
  LoopStart:
  iload_1
  bipush 101
  if_icmpge LoopEnd   // if (i >= 101) goto LoopEnd
  
  iload_1
  bipush 15
  irem
  ifne NotFizzBuzz    // if (i % 15 != 0) goto NotFizzBuzz
  
  // Print "FizzBuzz"
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "FizzBuzz"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  goto LoopIncrement
  
  NotFizzBuzz:
  // ... other conditions
  
  LoopIncrement:
  iinc 1 1            // i++
  goto LoopStart
  
  LoopEnd:
  return
}
```

### Increment Instruction

```jal
iinc index value    // Increment local variable by value
```

**Examples:**

```jal
iinc 0 1            // i++
iinc 1 -1           // j--
iinc 2 10           // k += 10
```

### Comparison

```jal
lcmp                // Compare two longs
fcmpg, fcmpl        // Compare floats (g/l on NaN)
dcmpg, dcmpl        // Compare doubles (g/l on NaN)
```

Returns: -1 (less), 0 (equal), 1 (greater)

## Switch Statements

### Table Switch

For contiguous integer cases:

```jal
tableswitch start {
  case_label_1,
  case_label_2,
  case_label_3
} default default_label
```

**Example from SwitchExample.jal:**

```jal
iload_1       // Load switch value
tableswitch 1 {
  Case1,      // For value 1
  Case2,      // For value 2
  Case3       // For value 3
} default Default

Case1:
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "One"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  goto Continue

Case2:
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "Two"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  goto Continue

Default:
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "Unknown"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V

Continue:
  return
```

### Lookup Switch

For sparse integer cases:

```jal
lookupswitch {
  case_value_1 : label_1,
  case_value_2 : label_2,
  default : default_label
}
```

**Example:**

```jal
iload_1       // Load switch value
lookupswitch {
  1 : LCase1,
  3 : LCase3,
  11 : LCase11,
  default : LDefault
}

LCase1:
  ldc "One (lookup)"
  // ... handle case
  goto End

LDefault:
  ldc "Other"
  // ... handle default
  goto End

End:
  return
```

## Exception Handling

### Try-Catch-Finally Structure

```jal
label: [~ startLabel
  , ExceptionType : catchLabel -> finallyLabel
] instruction;
```

### Components

- `[~ startLabel` - Begin protected region
- `, ExceptionType : catchLabel` - Exception handler (can have multiple)
- `-> finallyLabel` - Finally block (optional)
- `]` - End protected region

### Example

```jal
ProtectedBlock: [~ protectedStart,
    java/io/IOException: ioHandler -> cleanup, java/lang/Exception: generalHandler -> cleanup
] 
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "Executing protected code"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  goto afterCleanup
  
ioHandler:
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "IOException caught"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  goto cleanup
  
generalHandler:
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "Exception caught"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  
cleanup:
  getstatic java/lang/System -> out : Ljava/io/PrintStream;
  ldc "Finally block"
  invokevirtual java/io/PrintStream -> println (Ljava/lang/String;)V
  
afterCleanup:
  return
```

## Labels and Local Variable Information

### Labels

Labels mark positions that branching instructions can target:

```jal
labelName:
  // Instruction sequence
```

Used with:

- `goto`
- Conditional branches (`ifeq`, `if_icmpge`, etc.)
- Switch statements

### Local Variable Information

When storing to local variables, you can attach metadata:

```jal
istore_1 [I -> varName]
```

Components:

- Type descriptor (optional): Documents the expected type
- Label range (optional): Scope of validity
- Variable name: For debugging and readability

**Syntax:**

```jal
[TypeDescriptor? (~ labelName)? -> VariableName]
```

**Examples:**

```jal
public method (II)I {
  istore_1 [I -> firstArg]
  istore_2 [I -> secondArg]
  
  iconst_0
  istore_3 [I -> result]
  
  // ... method body
}
```

## Examples

### Example 1: HelloWorld (Simple Output)

```jal
public class HelloWorld (
  major_version=55,
  minor_version=0) {

  public static main([Ljava/lang/String;)V {
    // Print "Hello, World!"
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Hello, World!"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V

    // Return from main
    return
  }
}
```

**What this does:**

1. Load static field `System.out` (PrintStream)
2. Push string constant "Hello, World!"
3. Call `println()` method
4. Return (void)

### Example 2: FizzBuzz (Loops and Conditionals)

```jal
public class FizzBuzz (
  major_version=55,
  minor_version=0) {

  public static main([Ljava/lang/String;)V {
    // int i = 1;
    iconst_1
    istore_1

  LoopStart:
    // if (i >= 101) goto LoopEnd;
    iload_1
    bipush 101
    if_icmpge LoopEnd

    // if (i % 15 == 0)
    iload_1
    bipush 15
    irem
    ifne NotFizzBuzz

    // System.out.println("FizzBuzz");
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "FizzBuzz"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto LoopIncrement

  NotFizzBuzz:
    // else if (i % 3 == 0)
    iload_1
    iconst_3
    irem
    ifne NotFizz

    // System.out.println("Fizz");
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Fizz"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto LoopIncrement

  NotFizz:
    // else if (i % 5 == 0)
    iload_1
    iconst_5
    irem
    ifne PrintNumber

    // System.out.println("Buzz");
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Buzz"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto LoopIncrement

  PrintNumber:
    // System.out.println(i);
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    iload_1
    invokevirtual java/io/PrintStream->println(I)V

  LoopIncrement:
    // i++
    iinc 1 1
    goto LoopStart

  LoopEnd:
    return
  }
}
```

**Key patterns:**

- Local variable use: `istore_1`, `iload_1`
- Arithmetic: `irem` (modulo)
- Conditional branching: `ifne` (if not equal to zero)
- Loops: Using `goto` and labels
- Method calls: `invokevirtual`

### Example 3: BubbleSort (Arrays)

```jal
public class BubbleSort (
  major_version=55,
  minor_version=0) {

  public static main([Ljava/lang/String;)V {
    // int[] arr = {5, 3, 8, 4, 2};
    iconst_5
    newarray I              // Create int array of size 5
    
    // Initialize array elements using dup and iastore
    dup
    iconst_0
    iconst_5
    iastore                 // arr[0] = 5
    
    dup
    iconst_1
    iconst_3
    iastore                 // arr[1] = 3
    
    dup
    iconst_2
    bipush 8
    iastore                 // arr[2] = 8
    
    dup
    iconst_3
    iconst_4
    iastore                 // arr[3] = 4
    
    dup
    iconst_4
    iconst_2
    iastore                 // arr[4] = 2
    
    astore_1 [->targetArray]  // Store array reference

    // Outer loop: for (int i = 0; i < arr.length - 1; i++)
    iconst_0
    istore_2 [->i]

  OuterLoopStart:
    iload_2
    aload_1
    arraylength             // Get array length
    iconst_1
    isub
    if_icmpge OuterLoopEnd

    // Inner loop: for (int j = 0; j < arr.length - i - 1; j++)
    iconst_0
    istore_3 [->j]

  InnerLoopStart:
    iload_3
    aload_1
    arraylength
    iload_2
    isub
    iconst_1
    isub
    if_icmpge InnerLoopEnd

    // if (arr[j] > arr[j+1]) swap
    aload_1
    iload_3
    iaload                  // Load arr[j]
    
    aload_1
    iload_3
    iconst_1
    iadd
    iaload                  // Load arr[j+1]
    
    if_icmple NoSwap        // if arr[j] <= arr[j+1], skip swap

    // Swap logic here...
    // (detailed swapping code omitted for brevity)

  NoSwap:
    iinc 3 1                // j++
    goto InnerLoopStart

  InnerLoopEnd:
    iinc 2 1                // i++
    goto OuterLoopStart

  OuterLoopEnd:
    // Print sorted array
    iconst_0
    istore 5 [->k]

  PrintLoopStart:
    iload 5
    aload_1
    arraylength
    if_icmpge PrintLoopEnd

    getstatic java/lang/System->out:Ljava/io/PrintStream;
    aload_1
    iload 5
    iaload                  // Load arr[k]
    invokevirtual java/io/PrintStream->println(I)V

    iinc 5 1
    goto PrintLoopStart

  PrintLoopEnd:
    return
  }
}
```

**Key patterns:**

- Array creation: `newarray I`
- Array element access: `aaload`, `aastore`, `iaload`, `iastore`
- Array length: `arraylength`
- Nested loops with labels

### Example 4: TableSwitch (Switch Statement)

```jal
public class SwitchExample (
  major_version=55,
  minor_version=0) {

  public static main([Ljava/lang/String;)V {
    // int value = 2;
    iconst_2
    istore_1 [->value]

    // TableSwitch for value 1, 2, 3
    iload_1
    tableswitch 1 {
      Case1,
      Case2,
      Case3
    } default Default

  Case1:
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "One"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto LookupSwitchStart

  Case2:
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Two"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto LookupSwitchStart

  Case3:
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Three"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto LookupSwitchStart

  Default:
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Other"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V

  LookupSwitchStart:
    // Second switch with sparse values
    bipush 11
    istore_1

    iload_1
    lookupswitch {
      1 : LCase1,
      3 : LCase3,
      11 : LCase11,
      default : LDefault
    }

  LCase1:
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "One (lookup)"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto End

  LCase11:
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Eleven (lookup)"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V
    goto End

  LDefault:
    getstatic java/lang/System->out:Ljava/io/PrintStream;
    ldc "Other (lookup)"
    invokevirtual java/io/PrintStream->println(Ljava/lang/String;)V

  End:
    return
  }
}
```

**Key patterns:**

- `tableswitch`: For contiguous cases (efficient)
- `lookupswitch`: For sparse cases (flexible)
- Both require fall-through prevention with `goto`

## Best Practices

1. **Use Descriptive Labels**: Make branch targets clear
   ```jal
   OuterLoopStart:    // Good
   L1:                // Avoid
   ```

2. **Document Local Variables**: Use variable info for clarity
   ```jal
   istore_1 [I -> loopCounter]
   astore_2 [-> currentElement]
   ```

3. **Keep Methods Focused**: Each method should have a single responsibility

4. **Initialize Fields**: Use metadata for version compatibility
   ```jal
   public class MyClass (major_version=55, minor_version=0)
   ```

5. **Use Appropriate Instructions**: Choose optimized forms when possible
   ```jal
   iload_0         // Good: optimized
   iload 0         // Works but less efficient
   ```

6. **Document Complex Algorithms**: Use comments especially in loops
   ```jal
   // Bubble sort outer loop
   OuterLoopStart:
   // ...
   ```

## Summary

JAL provides a complete, human-readable syntax for expressing JVM bytecode. Key features include:

- **Class-level structure**: Metadata, fields, methods
- **Rich type system**: Primitives, objects, arrays
- **200+ instructions**: Stack, arithmetic, control flow, method calls
- **Exception handling**: Try-catch-finally constructs
- **Debug support**: Local variable information and labels

By combining these elements, JAL enables developers to write performant, low-level JVM code while maintaining
readability and structure.
